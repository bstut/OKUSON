\section{Motivation}
\frame{
{\large\textbf{Situation}}
\begin{itemize}
\item<1->
Anfängervorlesung mit über 1000 Studierenden
\item<2->
Wöchentliche Übungen mit Tutorien, 3 Klausuren
\item<3->
Etwa 20 Tutoren
\end{itemize}

\begin{itemize}
\item<4->[$\Rightarrow$] \glqq Klassischer Übungsbetrieb\grqq\ mit 4 schriftlichen 
Aufgaben/Woche nicht mehr möglich
\item<5->[$\Rightarrow$] Großer Verwaltungsaufwand
\end{itemize}
}

\frame{
{\large\textbf{Idee}}
\begin{itemize}
\item<2-> Teil der Aufgaben soll \glqq mechanisch\grqq\  zu korrigieren sein
(Multiple Choice, Abfrage von Rechenergebnissen)
\item<3-> Um Abschreiben zu erschweren, sollen \myemph{individuelle}
Aufgabenblätter ausgegeben werden (aus Pool von Aufgaben zusammengesetzt)
\item<4-> Hierzu soll die Verwaltung der Übungen über eine Webseite erfolgen:
\begin{itemize}
\item<5-> Anmeldung
\item<6-> Abholung der Übungsblätter
\item<7-> Abgabe der Multiple-Choice Aufgaben
\item<8-> Punktestand, Klausurergebnisse abfragen
\end{itemize}
\item<9-> Die Verwaltung soll möglichst automatisiert werden:
\begin{itemize}
\item<10-> Teilnehmer-, Punktestand-Verwaltung
\item<11-> Auswertung der Multiple-Choice Aufgaben
\end{itemize}
\end{itemize}
}

\frame{
{\large\textbf{Vorteile}}
\begin{itemize}
\item<1-> Tutoren werden entlastet, sie haben noch 2 statt 4 Aufgaben 
pro Teilnehmer und Woche zu korrigieren
\item<2-> Verwaltung wird entlastet: Übungsgruppeneinteilung, kein Drucken
von Aufgabenblättern, Punkteverwaltung, Scheinerstellung
\end{itemize}

\uncover<3->{\large\textbf{Nachteil (nur zu Beginn)}}
\begin{itemize}
\item<4-> Es müssen gute Ja/Nein, Multiple-Choice und Rechenaufgaben erfunden
werden, die mechanisch korrigiert werden können.
\end{itemize}


}

\section{Nutzung aus Teilnehmersicht}
\frame{
\large\textbf{Nutzung aus Teilnehmersicht}

Zur Teilnahme an Übungen, die mit \OKUSON\ verwaltet werden, wird benötigt:
\begin{itemize}
\item<2-> Internet-Anschluss
\item<3-> (graphik-fähiger) Webbrowser
\item<4-> Evtl. Möglichkeit zum Ausdrucken von PDF-Dateien
\end{itemize}

\uncover<5->{Insbesondere werden \myemph{kein} bestimmtes Betriebssystem, 
\myemph{keine} Browser-Plugins und
\myemph{keine} weitere externe Software benötigt.\\[2mm] }

\uncover<6->{\large\textbf{Ein Beispiel}

$\ldots$
}
}

\frame{
{\large\textbf{Aufgabentypen zur automatischen Auswertung}}
\begin{itemize}
\item<2-> \textbf{Single-Choice-Aufgaben:} Auswahl von Antworten, von denen
genau eine richtig ist
\item<3-> \textbf{Multiple-Choice-Aufgaben:} Auswahl von Antworten, von
denen die Teilmenge der richtigen zu finden ist
\item<4-> \textbf{Aufgaben mit einer eindeutigen, kurzen
Ergebniszeichenkette:} Etwa eine Zahl, die Ergebnis einer Rechnung ist, oder
ein Wort
\end{itemize}

\uncover<5->{{\large\textbf{Auswertung}}

Korrekt bewertete Antworten geben \myemph{einen Punkt}, falsche Antworten 
\myemph{einen Minuspunkt} und nicht beantwortete Fragen \myemph{Null
Punkte}. (Raten nützt nichts!)}
}

\section{Installation und Nutzung aus Veranstaltersicht}
\frame{
\large\textbf{Technische Voraussetzungen}
\begin{itemize}
\item<2-> UNIX/Linux System mit Internet-Anschluss
\item<3-> Installierte Software: Python, C-Compiler, (pdf)-\LaTeX,
ghostscript, netpbm, Texteditor, Webbrowser, PDF-Viewer  
(auf Linux-System meist alles vorhanden oder leicht zu installieren)
\end{itemize}

\uncover<4->{\large\textbf{Persönliche Voraussetzungen}}
\begin{itemize}
\item<5-> Installation und Start von Programmen
\item<6-> \LaTeX\ für Aufgabentexte
\item<7-> HTML Grundkenntnisse zum Editieren (Anpassen) von Webseiten
\end{itemize}
}

\frame{
{\large\textbf{Installation (ganz kurz)}}
\begin{itemize}
\item<2-> \OKUSON-Archiv von \\
{}\hspace{1cm}\texttt{http://www.math.rwth-aachen.de/\~{}OKUSON}\\
kopieren und auspacken
\item<3-> Konfigurations-Datei editieren, mindestens: Passwort, Vorlesungstitel,
Dozentenname, Semester, Kontaktadresse
\item<4-> \OKUSON-Server starten
\end{itemize}
}

\frame{
\uncover<1->{\large\textbf{Dokumentation und Lizenz}}
\begin{itemize}
\item<2-> Das \OKUSON-Archiv enthält eine ausführliche Dokumentation
(englisch). Erlaubt (hoffentlich) schnellen Einstieg und dient als
Referenzhandbuch.
\item<3-> \OKUSON\ enthält vollständigen dokumentierten Quellcode. 
%Für Programmierer ist es nicht sehr schwer, das System zu erweitern. 
\item<4-> \OKUSON\ ist \myemph{frei}, Copyright haben die Autoren, Nutzung
unter \myemph{GNU Public license}.
\item<5-> Die Autoren können leider keinen \glqq technischen Support\grqq\
bieten. Natürlich bemühen wir uns, auf Fragen zu antworten, Fehler zügig zu
beheben und Anregungen für Verbesserungen aufzugreifen.
\end{itemize}

}

\frame{
\uncover<1->{\large\textbf{Anpassung an lokale Gegebenheiten}}
\begin{itemize}
\item<2-> Über fortgeschrittenere Einträge der Konfigurationsdatei 
(mögliche Benutzer-IDs, spezielle \TeX-Makros, Funktion zur Notenberechnung,
Dateinamen für Daten,
$\ldots$)
\item<3-> Durch Anpassen der Webseiten:  
XHTML-Dateien mit  zusätzlichen Elementen, die dynamisch 
vom \OKUSON-Server ersetzt werden (Vorlesungsname, Personen-,
Übungsblatt-Daten, $\ldots$)
\item<4-> Neue Funktionalität durch Erweiterung der Programme
(wenn Programmierkenntnisse vorhanden)
\end{itemize}
}

\uncover<3->{\large\textbf{Eingabe von Aufgaben}}\\[2mm]

}
\uncover<5->{\textbf{Multiple-Choice Aufgaben:} auch in Dateien mit
einfacher XML-Syntax, die eigentlichen Aufgabentexte in \LaTeX\\
(Beispiel $\ldots$)
}
}

\section{Praktische Erfahrungen}
\frame{
{\large\textbf{Praktische Erfahrungen}}
\begin{itemize}
\item<2-> Vorgänger-Versionen  von uns seit drei Jahren benutzt
\item<3-> \OKUSON\ selbst jetzt im zweiten Semester (z.Z. für etwa
10 Vorlesungen an der RWTH)

\item<5-> Keinen Widerstand gegen  \glqq nur Web\grqq-Betrieb
